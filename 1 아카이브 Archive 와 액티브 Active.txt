1. 아카이브(Archive)와 액티브(Active)의 정의**

가장 먼저 작업 공간을 정돈해야 합니다.

| 구분 | 정의 | 포함될 대상 |
| :--- | :--- | :--- |
| **액티브(Active)** | **현재 개발 중이거나, '항해일지' 구축에 직접적으로 필요한 모든 자산.** 지금 당장 우리가 매일 만져야 하는 것들. | `ui/dev_a_vite`, `gumgang_0_5/backend`, `bridge`, `app`, `scripts`, `status`, `docs` |
| **아카이브(Archive)** | **역사적 가치는 있으나, 현재 개발에 직접 사용되지 않는 과거의 유산.** 금강의 발자취이자, 필요할 때 참고할 수 있는 '박물관'. | `ui/snapshots`, `ui/proto`, `ui/lc_app_recovered`, `QUARANTINE`, `LibreChat` (벤치마킹 자료이므로 참고용) |

#### **2. '금강 항해일지'를 위한 용어집**

직관적인 이해를 돕기 위해, 우리가 사용할 용어를 통일합니다.

| 공식 용어 | **우리의 용어 (금강식 표현)** | 설명 |
| :--- | :--- | :--- |
| Blueprint | **청사진** | 프로젝트의 전체적인 구조와 목표를 담은 최상위 설계도. (`docs` 폴더의 문서들) |
| Roadmap | **여정 지도** | '청사진'을 달성하기 위한 단계별 큰 그림. (예: 1단계-항해일지 구축, 2단계-콘텐츠 파이프라인 구현) |
| Milestone | **이정표** | '여정 지도' 위의 중요한 중간 목표 지점. (예: '항해일지 UI 초안 완성', '첫 자동 포스팅 성공') |
| Checkpoint | **점검 기록** | '이정표'로 나아가는 과정에서 남기는 구체적인 작업 증거 및 로그. (`status/checkpoints`의 파일들) |
| SSOT | **단일 진실 원천 (불변의 원칙)** | 한번 정해지면 거의 바뀌지 않는 프로젝트의 핵심 원칙과 설계 문서. (`docs` 폴더) |
| Dynamic Docs | **살아있는 문서 (유연한 계획)** | 상황에 따라 계속 업데이트되는 진행 상황, 계획, 로그. (`status` 폴더) |
